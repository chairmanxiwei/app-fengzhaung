# Web-Package - 网站封装APP平台

将任意网站快速封装为 Android APK 安装包，支持自定义应用名称、包名、图标、启动屏等，一键生成可安装的安卓应用。

## 功能特性

### 核心功能
- **网站封装**：输入网站 URL，自动生成 Android APK
- **自定义配置**：应用名称、包名、版本号、图标、启动屏图片
- **图标裁剪**：上传图标后支持在线裁剪，确保尺寸适配
- **启动屏设置**：自定义启动画面，防止应用启动黑屏
- **实时构建**：WebSocket 推送构建进度，实时查看构建状态

### 用户系统
- **注册/登录**：支持用户名（含中文）、手机号、密码注册登录
- **JWT 认证**：Token 持久化存储，服务重启不失效
- **每日配额**：登录用户每日 20 次构建，游客每日 3 次

### 构建管理（我的构建）
- **夹子系统**：左侧导航栏管理构建记录
  - **全部**：显示所有构建记录（含已删除）
  - **已删除**：仅显示已删除的记录
  - **未分类**：显示未归类的未删除记录
  - **自定义夹子**：用户可创建、删除自定义夹子
- **收藏功能**：标记常用构建，支持按收藏筛选
- **重命名**：修改构建记录的显示名称
- **移动**：将构建记录在夹子之间移动
- **备注**：为构建记录添加备注信息（最多 500 字）
- **软删除**：删除 APK 文件但保留历史记录，已删除记录自动归入"已删除"夹子
- **搜索**：按应用名称、备注内容搜索
- **排序**：按时间正序/倒序排列
- **状态筛选**：按完成/失败/构建中/收藏筛选

### 存储与下载
- **登录用户**：永久存储，无限次下载
- **游客**：24 小时临时存储，最多 5 次下载
- **下载令牌**：每次下载生成独立令牌，安全可控

### 界面交互
- **Toast 提示**：操作成功/失败即时反馈
- **按钮防重复**：操作锁机制，防止重复点击
- **自动刷新**：构建完成后自动刷新列表
- **响应式布局**：适配不同屏幕尺寸
- **时间精确到分钟**：构建记录显示 `YYYY-MM-DD HH:mm` 格式

## 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | Node.js + Express |
| 数据库 | SQLite (better-sqlite3) |
| 认证 | JWT (HS256) + bcryptjs |
| 前端 | 原生 HTML/CSS/JavaScript |
| 实时通信 | WebSocket (ws) |
| 构建 | Gradle + Android SDK |

## 快速开始

### 环境要求
- Node.js >= 18
- Android SDK（配置 `ANDROID_HOME` 环境变量）
- Gradle（项目内置 wrapper）

### 安装与运行

```bash
# 安装依赖
npm install

# 启动服务器
node server.js

# 访问
http://localhost:3000
```

### 环境变量

在 `.env.android` 中配置 Android 构建相关参数：

```
ANDROID_HOME=你的Android SDK路径
```

## 数据库表结构

- **users** - 用户表（用户名、密码哈希、手机号）
- **tasks** - 任务表（URL、应用名、包名、状态、进度、APK路径、收藏、备注、软删除标记等）
- **categories** - 分类表（用户自定义夹子）
- **quota_usage** - 配额使用记录（每日构建计数）
- **download_logs** - 下载日志
- **activity_logs** - 活动日志

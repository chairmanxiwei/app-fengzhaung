## 2. 数据库表结构设计

### 2.1 技术选型

| 对比项 | JSON 文件（当前） | better-sqlite3（方案） |
|--------|------------------|----------------------|
| 并发安全 | 无（全量读写） | ACID 事务 |
| 查询能力 | 全表扫描 | SQL 索引 |
| 性能 | O(n) 全量序列化 | O(1) 索引查询 |
| 数据完整性 | 无保障 | 外键约束 |
| 部署 | 零依赖 | npm install |
| 文件 | tasks.json 单文件 | data/webpackage.db 单文件 |

### 2.2 users — 用户账户

```sql
CREATE TABLE users (
  id            TEXT PRIMARY KEY,              -- UUID v4
  username      TEXT NOT NULL UNIQUE,          -- 3-20字符，字母数字下划线
  email         TEXT NOT NULL UNIQUE,          -- 邮箱地址
  password_hash TEXT NOT NULL,                 -- bcrypt 哈希 (cost=12)
  avatar_url    TEXT DEFAULT NULL,             -- 头像URL（可选）
  role          TEXT NOT NULL DEFAULT 'user',  -- user / admin
  status        TEXT NOT NULL DEFAULT 'active',-- active / suspended
  created_at    TEXT NOT NULL,                 -- ISO 8601
  updated_at    TEXT NOT NULL
);

CREATE UNIQUE INDEX idx_users_username ON users(username);
CREATE UNIQUE INDEX idx_users_email ON users(email);
```

**字段说明：**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PK | UUID v4，如 `u_550e8400e29b41d4a716446655440000` |
| username | TEXT | UNIQUE, NOT NULL | 3-20字符，仅 `[a-zA-Z0-9_]` |
| email | TEXT | UNIQUE, NOT NULL | 合法邮箱格式 |
| password_hash | TEXT | NOT NULL | bcrypt 哈希，60字符 |
| avatar_url | TEXT | 可选 | 用户头像链接 |
| role | TEXT | NOT NULL, DEFAULT 'user' | `user` 普通用户 / `admin` 管理员 |
| status | TEXT | NOT NULL, DEFAULT 'active' | `active` 正常 / `suspended` 停用 |
| created_at | TEXT | NOT NULL | 创建时间 ISO 8601 |
| updated_at | TEXT | NOT NULL | 更新时间 ISO 8601 |

### 2.3 sessions — 会话管理

```sql
CREATE TABLE sessions (
  id            TEXT PRIMARY KEY,              -- 随机 session ID
  user_id       TEXT NOT NULL,                 -- 关联用户
  token_hash    TEXT NOT NULL,                 -- JWT 的 SHA256 哈希
  ip_address    TEXT DEFAULT NULL,             -- 登录IP
  expires_at    TEXT NOT NULL,                 -- 过期时间
  created_at    TEXT NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE INDEX idx_sessions_user ON sessions(user_id);
CREATE INDEX idx_sessions_expires ON sessions(expires_at);
```

**字段说明：**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PK | 随机生成 |
| user_id | TEXT | FK → users.id | 所属用户 |
| token_hash | TEXT | NOT NULL | JWT 的 SHA256，用于撤销校验 |
| ip_address | TEXT | 可选 | 登录时的客户端IP |
| expires_at | TEXT | NOT NULL | 会话过期时间，默认7天 |
| created_at | TEXT | NOT NULL | 创建时间 |

### 2.4 categories — 安装包分类目录

```sql
CREATE TABLE categories (
  id            TEXT PRIMARY KEY,              -- UUID v4
  user_id       TEXT NOT NULL,                 -- 所属用户
  name          TEXT NOT NULL,                 -- 分类名称，1-30字符
  sort_order    INTEGER NOT NULL DEFAULT 0,    -- 排序权重（越小越靠前）
  created_at    TEXT NOT NULL,
  updated_at    TEXT NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  UNIQUE(user_id, name)                       -- 同用户下分类名唯一
);

CREATE INDEX idx_categories_user ON categories(user_id);
```

**字段说明：**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PK | UUID v4 |
| user_id | TEXT | FK → users.id | 所属用户 |
| name | TEXT | NOT NULL, UNIQUE(user_id,name) | 分类名称，1-30字符 |
| sort_order | INTEGER | DEFAULT 0 | 排序权重，0为最前 |
| created_at | TEXT | NOT NULL | 创建时间 |
| updated_at | TEXT | NOT NULL | 更新时间 |

### 2.5 tasks — 构建任务

```sql
CREATE TABLE tasks (
  id                TEXT PRIMARY KEY,          -- task_xxx
  user_id           TEXT DEFAULT NULL,         -- NULL=未登录用户
  category_id       TEXT DEFAULT NULL,         -- 所属分类
  url               TEXT NOT NULL,             -- 目标网址
  app_name          TEXT NOT NULL,             -- 应用名称
  package_name      TEXT NOT NULL,             -- 包名
  display_name      TEXT DEFAULT NULL,         -- 用户自定义显示名（重命名）
  icon_path         TEXT DEFAULT NULL,         -- 图标文件路径
  icon_original_name TEXT DEFAULT NULL,        -- 原始图标文件名
  status            TEXT NOT NULL DEFAULT 'QUEUED',
  progress          INTEGER NOT NULL DEFAULT 0,
  current_step      TEXT DEFAULT NULL,
  error_message     TEXT DEFAULT NULL,
  apk_path          TEXT DEFAULT NULL,         -- APK文件路径
  apk_size          INTEGER DEFAULT 0,         -- APK字节数
  version_name      TEXT NOT NULL DEFAULT '1.0',
  version_code      INTEGER NOT NULL DEFAULT 1,
  download_token    TEXT NOT NULL,             -- 下载令牌
  download_count    INTEGER NOT NULL DEFAULT 0,
  max_downloads     INTEGER NOT NULL DEFAULT 5,
  expires_at        TEXT DEFAULT NULL,         -- NULL=永久（登录用户）
  created_at        TEXT NOT NULL,
  updated_at        TEXT NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL,
  FOREIGN KEY (category_id) REFERENCES categories(id) ON DELETE SET NULL

CREATE INDEX idx_tasks_user ON tasks(user_id);
CREATE INDEX idx_tasks_category ON tasks(category_id);
CREATE INDEX idx_tasks_status ON tasks(status);
CREATE INDEX idx_tasks_expires ON tasks(expires_at);
CREATE INDEX idx_tasks_created ON tasks(created_at);
```

**字段说明：**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PK | `task_{timestamp}_{randomHex}` |
| user_id | TEXT | FK → users.id, 可选 NULL | NULL表示未登录用户创建 |
| category_id | TEXT | FK → categories.id, 可选 NULL | NULL表示未分类 |
| url | TEXT | NOT NULL | 目标网址 |
| app_name | TEXT | NOT NULL | 应用名称 |
| package_name | TEXT | NOT NULL | Android包名 |
| display_name | TEXT | 可选 | 重命名后的显示名，NULL时显示app_name |
| status | TEXT | NOT NULL | QUEUED/PREPARING/BUILDING/SIGNING/COMPLETED/FAILED |
| expires_at | TEXT | 可选 NULL | NULL=永久保存（登录用户），非NULL=过期时间 |

## 3. 用户认证流程设计

### 3.1 注册流程

```
用户填写表单
    │
    ▼
前端校验
  ├─ username: 3-20字符，仅 [a-zA-Z0-9_]
  ├─ email: 合法邮箱格式
  ├─ password: ≥8位，必须含字母+数字
  └─ confirmPassword: 与password一致
    │
    ▼
POST /api/auth/register
    │
    ├─ 用户名唯一性检查 → 400 USERNAME_TAKEN
    ├─ 邮箱唯一性检查 → 400 EMAIL_TAKEN
    ├─ bcrypt.hash(password, 12) → password_hash
    ├─ INSERT INTO users
    ├─ 自动登录（生成JWT）
    └─ 201 { success: true, token, user: {id, username, email} }
```

### 3.2 登录流程

```
用户输入 login(用户名或邮箱) + password
    │
    ▼
POST /api/auth/login
    │
    ├─ 查询用户 (WHERE username=? OR email=?)
    ├─ 用户不存在 → 401 INVALID_CREDENTIALS
    ├─ bcrypt.compare(password, hash)
    ├─ 密码错误 → 401 INVALID_CREDENTIALS
    ├─ 检查 status ≠ 'suspended' → 403 ACCOUNT_SUSPENDED
    ├─ 生成 JWT (HS256, payload: {userId, role}, expiresIn: 7d)
    ├─ INSERT INTO sessions (token_hash, expires_at)
    └─ 200 { success: true, token, user: {id, username, email, avatar} }
```

### 3.3 JWT 配置

| 配置项 | 值 |
|--------|-----|
| 算法 | HS256 |
| 密钥 | 环境变量 `JWT_SECRET`（启动时自动生成如未配置） |
| Payload | `{ userId, role, iat, exp }` |
| 有效期 | 7天 |
| 刷新策略 | 过期前24小时内可自动刷新 |
| 存储 | 前端 localStorage (`wp_token`) |

### 3.4 会话管理

```
每次请求:
  1. 从 Authorization: Bearer <token> 提取 JWT
  2. jwt.verify() 验证签名和有效期
  3. 查询 sessions 表确认 token 未被主动撤销
  4. 将 user 信息注入 req.user

登出:
  DELETE FROM sessions WHERE token_hash = SHA256(token)
  前端清除 localStorage

修改密码:
  删除该用户所有 sessions → 强制所有设备重新登录
```

### 3.5 认证中间件

| 模式 | 函数名 | 用途 | 行为 |
|------|--------|------|------|
| 必须登录 | `requireAuth` | 历史面板/删除/分类等 | 无Token返回401 |
| 可选登录 | `optionalAuth` | 创建任务/查询配额 | 无Token也放行，有Token则注入user |

## 4. 安装包存储与权限策略

### 4.1 登录 vs 未登录差异对照

| 能力 | 未登录 | 已登录 |
|------|--------|--------|
| **存储时长** | 24小时后自动删除 | **永久保存**（expires_at=NULL） |
| **每日配额** | **3次/天**（按IP） | **20次/天**（按用户ID） |
| **配额重置** | 每日0点 | 每日0点 |
| **下载次数** | 5次/任务 | **无限** |
| **历史记录** | sessionStorage（关闭即清空） | 数据库永久保存 |
| **删除权限** | 无 | 可删除（二次确认） |
| **分类管理** | 无 | 创建/修改/删除分类 |
| **重命名** | 无 | 可重命名（同分类下不重名） |
| **搜索筛选** | 无 | 按名称/分类/时间/状态 |
| **操作日志** | 无 | 完整记录 |
| **任务迁移** | 登录时可迁移临时任务 | — |

### 4.2 配额统计实现

```sql
-- 登录用户今日已用次数
SELECT COUNT(*) FROM quota_usage
WHERE user_id = ? AND date(used_at) = date('now', 'localtime');

-- 未登录用户今日已用次数（按IP）
SELECT COUNT(*) FROM quota_usage
WHERE ip_address = ? AND date(used_at) = date('now', 'localtime');
```

**配额检查流程：**

```
创建任务请求到达
    │
    ▼
判断是否登录
    │
    ├─ 已登录: 查询 quota_usage WHERE user_id=? AND date(used_at)=today
    │          已用 >= 20 → 429 QUOTA_EXCEEDED
    │
    └─ 未登录: 查询 quota_usage WHERE ip_address=? AND date(used_at)=today
               已用 >= 3 → 429 QUOTA_EXCEEDED
    │
    ▼
配额充足 → 创建任务 → INSERT quota_usage
```

### 4.3 过期清理策略

```
每小时执行清理任务:
  1. 查询 tasks WHERE expires_at < now AND expires_at IS NOT NULL
  2. 逐条处理:
     a. 删除 APK 文件 (task.apk_path)
     b. 删除图标文件 (task.icon_path)
     c. 记录 operation_logs (action='AUTO_CLEAN')
     d. DELETE tasks
  3. 清理 quota_usage 中超过7天的记录
  4. 清理 operation_logs 中超过30天的记录
  5. 清理 sessions 中已过期的会话

注意: 登录用户的任务 expires_at=NULL，永远不会被自动清理
```

## 5. 安装包管理功能

### 5.1 分类管理

| 操作 | API | 请求体 | 说明 |
|------|-----|--------|------|
| 创建分类 | `POST /api/categories` | `{name}` | 同用户下名称唯一 |
| 修改分类 | `PUT /api/categories/:id` | `{name, sortOrder}` | 修改名称或排序 |
| 删除分类 | `DELETE /api/categories/:id` | — | 该分类下任务移至"未分类" |
| 分类列表 | `GET /api/categories` | — | 含各分类下任务数量 |

**分类列表返回格式：**

```json
{
  "success": true,
  "data": [
    { "id": "cat_xxx", "name": "购物", "sortOrder": 0, "taskCount": 3 },
    { "id": "cat_yyy", "name": "社交", "sortOrder": 1, "taskCount": 5 },
    { "id": null, "name": "未分类", "taskCount": 2 }
  ]
}
```

### 5.2 安装包操作

| 操作 | API | 请求体 | 说明 |
|------|-----|--------|------|
| 列表查询 | `GET /api/tasks` | 查询参数 | 分页+筛选+搜索 |
| 重命名 | `PUT /api/tasks/:id/rename` | `{displayName}` | 同分类下不重名 |
| 移动分类 | `PUT /api/tasks/:id/move` | `{categoryId}` | 修改所属分类 |
| 删除 | `DELETE /api/tasks/:id` | — | 二次确认，删记录+文件 |
| 重新构建 | `POST /api/tasks/:id/rebuild` | — | 复用参数重新提交 |

```json
      "limit": 20,
      "total": 12,
      "totalPages": 1
    }
  }

```

### 5.4 重命名规则

- display_name 长度 1-30 字符
- 同一分类下 display_name 不能与同分类其他任务的 display_name/app_name 重复
- display_name 为 NULL 时，前端显示 app_name
- 重命名时记录操作日志

### 5.5 删除流程

```

    ▼

    │
    ▼

    │
    ▼

  1. 验证任务归属（user_id === req.user.id）
  2. 记录 operation_logs (action='DELETE', detail={appName, apkPath})
  3. 删除 APK 文件 (fs.unlink)
  4. 删除图标文件 (fs.unlink)
  5. DELETE FROM tasks WHERE id=?
  6. 返回 200 { success: true }
```

### 7.2 数据生命周期

| 事件 | 行为 |
|------|------|
| 创建任务 | 写入 sessionStorage + 服务端数据库 |
| 页面刷新 | 从 sessionStorage 恢复列表 |
| 关闭浏览器 | sessionStorage 自动清空，服务端数据24小时后过期清理 |
| 登录成功 | 提示迁移临时任务 → 调用迁移API → 清空 sessionStorage |

### 7.3 登录后迁移

```

    │
```

## 10. 数据安全与隐私保护

### 10.1 密码安全

| 措施 | 说明 |
|------|------|
| 存储方式 | bcrypt (cost factor=12)，不可逆 |
| 传输安全 | HTTPS 加密（生产环境必须） |
| 密码策略 | ≥8位，必须含字母+数字，禁止常见弱密码 |
| 密码不可见 | API 永不返回 password_hash |

### 10.2 Token 安全

| 措施 | 说明 |
|------|------|
| 签名密钥 | 从 `JWT_SECRET` 环境变量读取，启动时自动生成 |
| 哈希存储 | sessions 表存储 token 的 SHA256，支持撤销 |
| 强制登出 | 修改密码后删除所有 sessions |
| 传输方式 | Authorization header，不通过 URL 传递 |

### 10.3 数据隔离

| 措施 | 说明 |
|------|------|
| 用户隔离 | 查询/操作只能访问自己的数据（WHERE user_id=?） |
| 未登录隔离 | 未登录任务仅通过 downloadToken 访问 |
| 管理员预留 | admin 角色可查看所有数据（预留，暂不实现） |

### 10.4 输入验证

| 输入项 | 验证规则 |
|--------|---------|
| username | `/^[a-zA-Z0-9_]{3,20}$/` |
| email | 标准邮箱正则 + 域名验证 |
| password | 长度≥8 + 包含字母 + 包含数字 |
| categoryName | 1-30字符，同用户唯一 |
| displayName | 1-30字符，同分类唯一 |
| 所有输入 | 入库前做 HTML 实体转义 |

### 10.5 下载令牌安全

- 使用 `crypto.randomBytes(24)` 生成，48位十六进制
- 比较时使用 `crypto.timingSafeEqual` 防止时序攻击
- 登录用户下载不计入 download_count

## 14. 附录：数据迁移方案

### 14.1 tasks.json → SQLite 迁移

```
迁移脚本: db/migrations/migrate-json-to-sqlite.js

步骤:
  1. 读取 data/tasks.json
  2. 逐条 INSERT INTO tasks
  3. 登录用户的任务: expires_at = NULL (永久)
  4. 未登录用户的任务: 保持原 expires_at
  5. 迁移完成后将 tasks.json 重命名为 tasks.json.bak
  6. 记录迁移日志

字段映射:
  JSON字段          → SQLite字段           转换规则
  id               → id                   直接映射
  (无)             → user_id              NULL (旧数据无用户关联)
```
